Web Storage no JavaScript

localStorage e sessionStorage — armazenamento de dados no navegador sem servidor

Característica localStorage sessionStorage
Duração dos dados Persiste indefinidamente (até o usuário limpar) Apagado ao fechar a aba ou o navegador
Escopo Compartilhado entre todas as abas da mesma origem Isolado por aba — cada aba tem o seu próprio
Capacidade típica ~5 MB por origem ~5 MB por origem
Enviado ao servidor? Não (diferente de cookies) Não (diferente de cookies)
Acessível via JS? Sim — window.localStorage Sim — window.sessionStorage
Tipo dos dados Somente strings (use JSON para objetos) Somente strings (use JSON para objetos)
Uso típico Preferências, tema, token de sessão longa Dados de formulário em progresso, carrinho temporário

1. localStorage — setItem e getItem

Salve e recupere strings simples no localStorage. Recarregue a página e veja que os dados continuam lá.

—

2. sessionStorage — setItem e getItem

Funciona exatamente como o localStorage, mas os dados somem ao fechar esta aba. Abra numa nova aba e veja que os dados não são compartilhados.

—

3. removeItem e clear — localStorage

Use removeItem para apagar uma chave específica e clear para limpar todo o localStorage de uma vez. Cuidado: clear() afeta todos os dados salvos pela sua aplicação.

—

4. Iterando o localStorage (length + key)

localStorage.length informa quantos itens existem. localStorage.key(i) retorna a chave na posição i. Juntos permitem listar tudo que está salvo.

Clique em "Listar" para inspecionar

5. Armazenando Objetos com JSON.stringify / JSON.parse

O Web Storage só aceita strings. Para persistir objetos completos usamos JSON.stringify ao salvar e JSON.parse ao ler — o mesmo padrão visto na Aula 21, agora aplicado ao localStorage.

—

6. sessionStorage — Rascunho por Aba

Simula um rascunho automático de formulário. Os dados são salvos a cada digitação (evento input) e restaurados ao recarregar a página — mas somem ao fechar a aba.

—

7. Evento storage — Sincronização entre Abas

O evento window.addEventListener('storage', ...) dispara quando o localStorage é modificado por outra aba da mesma origem. Abra esta página em duas abas e salve um valor na Seção 1 — observe o log nesta aba.

Importante: o evento NÃO dispara na aba que fez a alteração — só nas outras.
