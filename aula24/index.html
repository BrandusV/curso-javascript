Callbacks, Promises, async/await, fetch e a Event Loop — quando e como usar cada um

1. Callbacks
ES1 — 1997
Funções passadas como argumento para serem chamadas depois. Funcionam, mas geram "callback hell" ao aninhar.
2. Promises
ES6 — 2015
Objeto que representa um valor futuro. Resolve o aninhamento com .then() e .catch() encadeados.
3. async / await
ES8 — 2017
Sintaxe sobre Promises que permite escrever código assíncrono com aparência síncrona. Mais legível e fácil de depurar.
4. Event Loop
Mecanismo do motor JS
O que torna tudo possível. Move tarefas entre Call Stack, Web APIs e a Fila de Callbacks automaticamente.

1. Callbacks — o padrão original

Uma função é passada como argumento e chamada quando a operação termina. Simples para um nível, mas vira "callback hell" ao aninhar etapas dependentes.

—

2. async / await — sintaxe moderna

async marca a função como assíncrona. await pausa a função até a Promise resolver — sem bloquear o restante da página.

—

3. try / catch / finally com async

Erros em await são capturados com try/catch — sem precisar de .catch(). finally sempre executa, ideal para esconder indicadores de carregamento.

—

4. fetch() — requisições HTTP reais

fetch() faz requisições HTTP e retorna uma Promise. Dois await são necessários: um para a resposta chegar e outro para ler o corpo JSON. Usamos a API pública JSONPlaceholder.

—

5. Sequencial vs Paralelo com await

await um por um = sequencial (soma dos tempos). Promise.all + await = paralelo (tempo do mais lento). Use paralelo sempre que as operações forem independentes entre si.

6. A Event Loop — como o JS assíncrono funciona por dentro

O JavaScript é single-thread: executa uma coisa por vez. A Event Loop coordena três áreas: a Call Stack (execução atual), as Web APIs (tarefas em segundo plano) e a Fila de Callbacks (prontos para executar). Clique para animar o fluxo passo a passo.
